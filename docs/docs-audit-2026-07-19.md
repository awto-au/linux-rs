## Pattern flagged: in-place "correction" layering

Three docs append a dated correction notice on top of original now-wrong
text instead of stating the corrected content directly:
`docs/8250-tier-c-blocker-2026-07-18.md` ("Update (2026-07-18, later same
day):"), `docs/phase0-evals.md` ("Superseded 2026-07-18:"), and
`docs/phase1-census-v1.md` (v1.0→v1.1 inline correction). This is a bad
pattern going forward — future docs (and edits to these) should state the
correct conclusion directly and drop the archaeology of what used to be
wrong. Not recommending a rewrite of these three now (the archaeology is
part of what makes them a genuine record of a reversed finding — see the
tmpfs pair, which is the same situation done as two separate docs instead
of one self-correcting one) — but no new doc should use this shape, and any
future edit to these three is a good opportunity to clean it up.
